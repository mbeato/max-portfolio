    ```bash
    npx lighthouse http://localhost:3000/work/tonos --only-categories=performance --form-factor=mobile --output=json --output-path=./lighthouse-tonos.json --chrome-flags="--headless=new --no-sandbox"
    ```

    **Step 4 — Evaluate results against targets (D-08):**
    - If home page score >= 90 AND tonos score >= 90 AND LCP < 2500ms: PASS — no further performance work needed.
    - If score < 90 on either page, apply fixes in D-06 priority order:

    **Fix priority 1 — Font loading:** Check if LCP element is text and Lighthouse flags "Ensure text remains visible during webfont load". Space Grotesk is loaded via next/font with display: 'swap' — this should already be optimal. If flagged, verify the font @font-face is being inlined in <head> by Next.js (it should be automatic in Next.js 15).

    **Fix priority 2 — Canvas deferred init:** Check if Lighthouse flags long tasks related to canvas. The canvas uses useEffect (non-blocking). If TBT is high, check if d3-contour or simplex-noise are bundled in the main chunk. If so, add dynamic import:
    ```typescript
    // In TopoCanvas.tsx or useTopoAnimation.ts — dynamic import if needed
    const { contours } = await import('d3-contour');
    ```

    **Fix priority 3 — Image optimization:** Plan 01 already added `sizes` props. If Lighthouse still flags "Properly size images", check if images need explicit `width`/`height` hints or if `priority` is set on above-fold images only (Me.jpg should have priority, others should not).

    **Fix priority 4 — JS bundle:** If TBT > 300ms, install and run bundle analyzer:
    ```bash
    npm install --save-dev @next/bundle-analyzer
    ```
    Add to next.config.ts temporarily, identify large chunks, apply code splitting.

    **Step 5 — Re-run Lighthouse after any fixes to confirm targets met.**

    Clean up: Stop the next start process. Delete lighthouse JSON files (or keep for reference).

    Record final scores in the summary.
